How to launch the nToklo account & application creation process

Code to include in your extension

Styles

The ntoklo.css file should be linked to in the <head> of your page.

<head>

<link href="ntoklo.css" rel="stylesheet"
media="screen, projection" type="text/css" />

Buttons and frame container to launch the account & application creation process

You must include the following markup in your extension panel. When the user clicks on either button, an iFrame with the registration / login form will be displayed in the ntIFrameWrapper.

In the HTML of your extension settings panel

<a class="btn no_account" href="#" id="ntLaunchRegister">
I don't have an nToklo account </a>
<a class="btn account" href="#" id="ntLaunchLogin">
I already have an nToklo account </a>
<div id="ntIFrameWrapper"></div>
<script type="text/javascript"> window.ntParams = { "p" : "Platform", "f" : "User's first name", "l" : "User's last name", "e" : "User's email", "n" : "Store name", "d" : "Store URL" } </script>
<script src="ntoklo.js" type="text/javascript"></script>

Javascript configuration values (in window.ntParams)

The values in the window.ntParams javascript object shown in red above should be replaced with values from the store account details.

See an example

Using the following window.ntParams object, you can see the values are passed through into the registration and log in forms.

Test window.ntParams object

<script type="text/javascript"> window.ntParams = { "p" : "Platform", // this will be set by the drop down "f" : "Jane", "l" : "Smith", "e" : "janesmith@mystore.com", "n" : "My Store", "d" : "www.mystore.com" } </script>
<script src="ntoklo.js" type="text/javascript"></script>

Valid values for window.ntParams.p are:

  • magento
  • bigcommerce
  • wpecommerce
  • oscommerce
  • zencart
  • shopify
  • presta
  • virtuemart

These buttons will appear in the settings / admin panel of your extension
